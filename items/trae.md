# Trae

## 一句话总结

一个 AI 原生 IDE / 编码工作台，提供 IDE 模式与更自主的 SOLO / Builder 模式，支持多 Agent 协作、上下文理解和从需求到交付的开发流程。

## 它解决什么问题

- 传统 AI 编辑器往往只把补全或聊天嵌进 IDE，难以真正承接完整开发流程
- 开发者希望在“自己主导编辑”和“交给 Agent 自主推进”之间灵活切换
- 全球版和中国版 AI IDE 常在模型、定位和落地方式上分化，用户不容易一眼看清产品边界

## 适合谁

- 想把 AI 原生工作流放进日常 IDE 的开发者
- 希望在传统编辑器体验和更自主的编码 Agent 之间切换的人
- 关注中国版与全球版 AI IDE 产品演进的用户和团队

## 核心能力

- 双工作模式：既有更贴近 IDE 的协作模式，也有更自主的 SOLO / Builder 式开发模式
- 多 Agent 能力：强调内置 Agent 与自定义 Agent 协作处理更复杂任务
- 全流程开发：从理解需求、规划、执行到交付，覆盖更完整的软件开发链路
- 区域化版本：全球版强调 SOLO 与多 Agent 协作，中国版强调中文场景和本地模型集成

## 典型使用场景

- 在 AI IDE 里完成从需求到代码交付的连续开发流程
- 在常规编辑工作和更自主的 Agent 执行之间快速切换
- 对比全球版与国内版 AI IDE 的能力差异，选择更匹配团队环境的版本

## 为什么值得关注

- 它不是单点补全工具，而是往“完整 AI 开发工作台”方向走
- 全球版和国内版都在强调更强的自主开发模式，这点和普通 AI 编辑器明显不同
- 对观察 AI IDE 产品形态分化很有价值，尤其是国际版与中国版在模型和定位上的差别

## 类似项目

- [CodePilot](codepilot.md) - 更偏多模型 Agent 桌面客户端，而 Trae 更像 AI 原生 IDE / 编辑器产品。
- [Tabnine](tabnine.md) - 更偏补全与团队可控编码助手，而 Trae 更强调 Agent 模式和完整开发流程。
