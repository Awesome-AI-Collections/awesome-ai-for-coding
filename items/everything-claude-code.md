# Everything Claude Code

## 一句话总结

一个面向 Claude Code、Codex、OpenCode 等 agent harness 的高性能工程化系统，整合 skills、hooks、memory、持续学习、安全扫描和 token optimization，帮助团队长期稳定地用 AI 写软件。

## 它解决什么问题

- 只装几个 prompt 或 config，通常不足以支撑长期稳定的 AI 编码工作流
- 会话记忆、性能优化、安全检查和经验沉淀如果彼此割裂，AI 工程系统很容易失控
- 团队常同时使用多个 harness，但难以复用一套统一规则和基础设施

## 适合谁

- 想把 Claude Code 调成长期可用工程系统的开发者
- 需要跨多种 agent harness 共享规则、技能和 hook 的团队
- 重视性能优化、安全与持续学习闭环的 AI 工程用户

## 核心能力

- Harness 基础设施：覆盖 Claude Code、Codex、Cursor、OpenCode、Gemini 等多环境
- Hook 与规则系统：通过 hooks、rules、skills 和命令 shim 组织统一行为层
- Memory 与学习：强调会话记忆持久化、模式提炼和持续学习
- 性能与安全：同时把 token optimization、安全扫描和验证流程纳入日常工作流

## 典型使用场景

- 把 Claude Code 从“个人配置”升级成跨项目的工程系统
- 在多个 agent harness 之间共享同一套规则、技能和运行策略
- 为 AI 原生团队建立性能、安全、记忆和复盘一体化底座

## 为什么值得关注

- 它不是单一技巧仓，而是“agent harness operating system”式的整合思路
- 覆盖面很广，适合观察 AI 工程化实践如何从配置演化成平台
- 对已经不满足于“单个提示词更好用”的团队特别有参考价值

## 类似项目

- [Superpowers](superpowers.md) - 同样强调技能与工程纪律，但 Superpowers 更偏方法论和执行流程；Everything Claude Code 更偏底层运行系统与跨 harness 基础设施。
- [gstack](gstack.md) - 同样围绕 Claude Code 打造完整工作流，但 gstack 更偏高层软件工厂；Everything Claude Code 更偏底层性能、hooks 和系统化配置。
