# Cursor

## 一句话总结

一个 AI 原生代码编辑器与开发工作台，提供智能体、代码审查、Tab 自动补全、云端 Agent 和多模型选择，支持从协作编辑到更自主的端到端开发。

## 它解决什么问题

- 传统编辑器即使接入 AI，也常常只停留在聊天或补全层，难以覆盖完整开发流程
- 开发者希望在“局部辅助”与“更自主的 Agent 执行”之间自由切换，而不是被固定交互模式限制
- 团队在引入 AI 编码时，不只需要生成代码，还需要代码审查、上下文理解和跨界面协作能力

## 适合谁

- 想把 AI 原生能力直接放进日常编辑器的开发者
- 需要在补全、定向编辑和更自主 Agent 模式之间切换的工程团队
- 关注全球版与中文官网产品演进的 Cursor 用户

## 核心能力

- 多种 AI 交互模式：同时覆盖 Tab 补全、定向编辑、代码审查和更自主的智能体模式
- 全代码库理解：强调对大型复杂代码库的索引、理解和上下文调用
- 多模型工作流：支持在不同前沿模型之间切换，按任务选择更合适的模型
- 跨界面协作：除了 IDE，本身也在往 GitHub、云端智能体等更完整工作台方向扩展

## 典型使用场景

- 在日常 IDE 工作流中用 Tab、编辑和 Agent 模式协同完成开发
- 让 AI 处理局部代码修改、PR 审查和更完整功能实现
- 对比不同 AI IDE 产品时，把 Cursor 作为主流代表产品之一观察和选型

## 为什么值得关注

- 它是 AI IDE 赛道里最有代表性的产品之一，影响了很多后续产品形态
- 不只做“代码补全”，而是在往完整 AI 开发工作台持续演进
- 中文官网也保持了较完整的产品说明，便于直接面向中文读者整理

## 类似项目

- [Trae](trae.md) - 同样强调 AI IDE 与更自主的 Agent 模式，但 Trae 在全球版和中国版的产品分化更明显。
- [Tabnine](tabnine.md) - 更偏编码助手与企业可控能力，而 Cursor 更强调一体化 AI IDE 体验和更强的 Agent 工作流。
