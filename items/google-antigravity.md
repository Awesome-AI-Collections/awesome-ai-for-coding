# Google Antigravity

## 一句话总结

一个面向 agent-first 时代的 AI IDE 与开发平台，强调跨编辑器、终端、浏览器的多 Agent 协作与任务编排，提供上下文感知的智能编辑体验。

## 它解决什么问题

- 多数 IDE 仍然是“人写代码为主”，而 agent-first 工作流需要更系统的编排与监控
- 多 agent 会话在不同工具间分散，难以集中管理与验证
- 开发者希望把编辑、验证、协作和 agent 管理整合到一个统一平台

## 适合谁

- 想要 agent-first AI IDE 的开发者
- 需要在多个工作区和多 agent 会话间集中管理的团队
- 关注跨工具协作与验证闭环的工程组织

## 核心能力

- AI IDE 核心：强调自动补全、自然语言指令与上下文感知 agent
- 任务级编排：以任务为中心监控 agent 活动与关键产出
- 跨界面协作：同步管理编辑器、终端与浏览器中的 agent
- 体验与可视化：通过统一的“mission control”视图管理并行会话

## 典型使用场景

- 让多个 agent 并行处理前端、后端与运维任务，再统一回收验证
- 在多工作区环境下集中管理 agent 与工程资产
- 需要一个“IDE + agent 管理平台”合体体验的团队

## 为什么值得关注

- 它把 agent-first 叙事放在核心位置，代表一种新的 IDE 形态
- 强调跨界面协同与验证闭环，适合复杂工程团队
- 如果后续与 Google 生态更深度联动，可能成为重要平台级选项
