# r.jina.ai

## 一句话总结

一个通过 URL 前缀把任意网页转成纯 Markdown 的超轻量入口，适合给 Claude Code、OpenClaw 等 Agent 做网页正文抓取和快速阅读。

## 它解决什么问题

- 网页正文常常夹杂大量 HTML、脚本和样式，对 LLM 很不友好
- 临时抓取网页时，开发者不一定想装浏览器扩展、爬虫库或复杂代理
- 想让 Agent 快速学习某篇文章、文档或帖子时，最需要的是一个低摩擦入口

## 适合谁

- 想让 Claude Code 或 OpenClaw 直接读网页正文的开发者
- 做信息整理、技术学习和网页摘要的研究用户
- 需要一个随手可用的 Markdown 抽取入口，而不是一整套抓取工程的人

## 核心能力

- URL 前缀转换：把真实 URL 拼到前缀后面即可读取
- Markdown 输出：比直接喂原始 HTML 更适合 Agent 阅读和总结
- 低摩擦接入：适合临时调用、脚本串联和命令行工作流
- 通用网页读取：对文章、文档、帖子等内容型页面尤其有效

## 典型使用场景

- 给 OpenClaw 快速喂一篇网页文章做摘要或结构分析
- 在 Claude Code 里临时抓技术博客、X 帖子或文档正文
- 当成浏览器自动化前的轻量抓取第一步

## 为什么值得关注

- 它把“网页转可读文本”这件事压缩成极低门槛的 URL 习惯
- 在 Agent 工作流里，这种简单工具往往比重量级方案更高频
- 很适合作为你的数据采集工具栈里的第一道入口

## 类似项目

- [OpenCLI](opencli.md) - 更适合需要真实浏览器交互和多宿主整合的复杂自动化
- [bb-browser](bb-browser.md) - 更适合需要登录态和真实浏览器上下文的平台读取
