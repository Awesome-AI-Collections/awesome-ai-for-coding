# HumanLayer / CodeLayer

## 一句话总结

HumanLayer 团队推出的 CodeLayer 开源 AI IDE，围绕 Claude Code 提供键盘优先、多会话并行、worktree 与云 worker 编排，以及 advanced context engineering 工作流。

## 它解决什么问题

- Claude Code 在复杂代码库和团队场景里，缺少更强的 IDE 级编排与可视化工作台
- 多会话并行、云端 worker 和复杂上下文管理如果纯靠终端，门槛较高
- 团队希望把 context engineering 变成可操作的 IDE 工作流，而不是抽象理念

## 适合谁

- 想把 Claude Code 提升成团队级 AI IDE 体验的开发者
- 需要并行多会话和复杂代码库工作流的工程团队
- 关注 context engineering 与 agent orchestration 的 AI 原生组织

## 核心能力

- AI IDE 体验：以键盘优先方式组织 Claude Code 工作流，而不是只做一个命令行外壳
- 并行会话编排：支持多 Claude 会话、worktree 与远程 cloud worker 并行推进
- Context engineering：强调在复杂代码库里保持上下文质量，而不是任由 agent 漫游
- 团队可扩展性：从个人笔记本到团队协作场景逐步放大

## 典型使用场景

- 在大代码库里并行跑多个 Claude Code 会话并集中观察产出
- 给团队建立更统一、更高控制力的 Claude Code IDE 层
- 把“context engineering”从 talk 和方法论变成实际开发界面和工作流

## 为什么值得关注

- 它抓住的是 Claude Code 走向团队化和 IDE 化这一步
- HumanLayer 在 context engineering 叙事上的影响力，让这个产品更值得跟踪
- 如果 CodeLayer 成熟，它会是 Claude Code 周边非常关键的一层基础设施

## 类似项目

- [Conductor](conductor.md) - 同样关注并行编码 agent 和 worktree 流程，但 Conductor 更聚焦轻量编排；HumanLayer / CodeLayer 更强调 IDE 和 context engineering。
- [Acepe](acepe.md) - 同样是多 agent 工程工作台，但 Acepe 更强调权限、checkpoint 和 PR 流程；HumanLayer / CodeLayer 更强调 Claude Code 与上下文工程。
