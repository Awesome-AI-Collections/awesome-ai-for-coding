# Nezha

## 一句话总结

一个 Agent-First 的轻量桌面应用，用来在多个项目间并行运行 Claude Code 和 Codex，并把任务跟踪、终端、会话回放、代码浏览和 Git 工作流整合到同一个界面里。

## 它解决什么问题

- 当 Claude Code 和 Codex 并行跑多个项目时，终端、任务状态、代码浏览和 Git 很容易散落在不同工具里
- 传统 IDE 还是以人工编码为中心，不一定适合“多 Agent 并行 + 多项目切换”的工作方式
- 开发者想用更轻、更快的桌面工具承接 AI 编程工作流，而不是总开着笨重 IDE

## 适合谁

- 想并行运行 Claude Code 和 Codex 的开发者
- 需要在多个项目和任务之间快速切换的 AI 编程重度用户
- 希望用更轻量桌面工具替代笨重 IDE 做 agent 工作流管理的人

## 核心能力

- 多项目工作区：支持在多个代码库和任务之间快速切换
- 会话与任务可视化：自动关联 Claude Code / Codex 会话，支持任务状态跟踪与会话回放
- 终端与代码浏览：内置实时终端、代码和 Markdown 编辑器
- 原生 Git 集成：支持 diff、日志、分支管理和 AI 辅助提交信息

## 典型使用场景

- 同时推进多个 vibe coding 任务，并在不同项目间快速切换
- 用一个桌面应用承接 Claude Code / Codex 的任务管理、审查和提交流程
- 在不打开完整 IDE 的情况下完成多 Agent 工作流闭环

## 为什么值得关注

- 方向很明确：为 AI 编程时代重新做一套更轻量的桌面工作台
- 它把任务生命周期和会话可视化放在很前面，不只是再包一层终端
- 对平时就用 Claude Code / Codex 多开的人来说，这类专用桌面层很有代表性

## 类似项目

- [Superconductor](superconductor.md) - 同样面向并行 coding agents，但 Superconductor 更强调 worktree 和原生性能；Nezha 更强调任务可视化和轻量桌面工作流。
- [MindFS](mindfs.md) - 更偏浏览器远程访问本地 Agent，而 Nezha 更像本机桌面端的多项目工作台。
