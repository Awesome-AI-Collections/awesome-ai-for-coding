# XCrawl

## 一句话总结

一个面向网页与站点数据获取的托管 API 服务，已经提供 OpenClaw 接入文档，适合不想自己长期维护浏览器自动化和反爬细节的团队。

## 它解决什么问题

- 自建抓取栈往往要长期处理浏览器环境、风控适配、稳定性和出口问题
- 当需求重点在“拿到数据”而不是“研究抓取底层”时，自建方案成本过高
- OpenClaw 这类 Agent 工作流想快速接入外部网页数据时，需要更省心的服务层

## 适合谁

- 想直接用托管 API 做网页与站点数据获取的开发者
- 不想维护整套抓取基础设施的自动化工程团队
- 正在给 OpenClaw 配外部数据入口的用户

## 核心能力

- 托管式抓取：把复杂抓取基础设施封装成服务接口
- OpenClaw 兼容：官方文档已提供 OpenClaw 接入说明
- 降低维护成本：更适合不想自己长期维护浏览器和反爬细节的团队
- 服务化使用：适合接入现有工作流，而不是从零搭底层

## 典型使用场景

- 给 OpenClaw 增加一个省心的数据采集服务后端
- 在原型阶段先用托管服务验证需求，再决定是否自建
- 把需要稳定输出的网页数据采集能力交给外部服务层

## 为什么值得关注

- 它和本轮调研里那些开源工具形成互补：一个偏自建，一个偏托管
- 对很多团队来说，真正稀缺的不是“能不能抓”，而是“能不能省心稳定地抓”
- 已经有 OpenClaw 文档意味着它不是泛泛而谈，而是在主动靠近 Agent 场景
